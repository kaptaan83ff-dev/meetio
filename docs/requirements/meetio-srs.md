> **Version:** 0.1 (placeholder)
> **Status:** Draft / Not yet authored
> **Last Updated:** May 16, 2026

This repository currently uses the PRD + TRD as the primary requirement sources:

- `docs/requirements/meetio-prd.md` — product requirements (what/why)
- `docs/requirements/meetio-trd.md` — technical requirements (how/constraints)
- `docs/requirements/meetio-api-spec.md` — endpoint contracts
- `docs/requirements/meetio-db-schema.md` — schema/contracts for persistence

When an SRS is added, it should focus on:

- precise, testable functional requirements (FRs)
- non-functional requirements (NFRs) and acceptance criteria
- traceability mapping to PRD sections and API endpoints
